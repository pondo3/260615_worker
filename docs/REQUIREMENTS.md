# REQUIREMENTS.md - 기능 요구사항 명세서

## 1. 인증 기능 (로그인/회원가입)

### 회원가입
- [ ] 이메일 + 비밀번호로 회원가입
- [ ] 이름 입력 필수
- [ ] 비밀번호는 bcrypt로 암호화하여 DB 저장
- [ ] 이미 가입된 이메일이면 에러 메시지 표시
- [ ] 비밀번호 최소 8자 이상

### 로그인
- [ ] 이메일 + 비밀번호로 로그인
- [ ] 잘못된 정보 입력 시 에러 메시지
- [ ] 로그인 성공 시 대시보드로 이동
- [ ] 로그인 상태는 세션으로 유지

### 로그아웃
- [ ] 로그아웃 버튼 클릭 시 세션 종료
- [ ] 로그인 페이지로 이동

### 접근 제어
- [ ] 로그인하지 않으면 대시보드 접근 불가
- [ ] 로그인하지 않으면 자동으로 로그인 페이지로 이동
- [ ] 본인 데이터만 볼 수 있음 (다른 사용자 데이터 접근 불가)

## 2. 할일 관리 기능

### 할일 등록
- [ ] 제목 (필수)
- [ ] 상세 내용 (선택)
- [ ] 날짜 (필수, 기본값: 오늘)
- [ ] 마감 시간 (선택)
- [ ] 중요도: 높음 / 보통 / 낮음
- [ ] 긴급도: 높음 / 보통 / 낮음
- [ ] 상태: 대기 / 진행중 / 완료 / 보류 / 취소

### 할일 분류 (아이젠하워 매트릭스)
| 분류 | 조건 |
|------|------|
| 1순위: 지금 당장 해야 함 | 중요도 높음 + 긴급도 높음 |
| 2순위: 계획적으로 해야 함 | 중요도 높음 + 긴급도 낮음 |
| 3순위: 위임할 수 있음 | 중요도 낮음 + 긴급도 높음 |
| 4순위: 나중에 해도 됨 | 중요도 낮음 + 긴급도 낮음 |

### 할일 보기
- [ ] 오늘 할일 목록
- [ ] 내일 할일 목록
- [ ] 지난 미완료 할일 목록 (기한 지났지만 완료 안 됨)
- [ ] 완료한 할일 목록
- [ ] 전체 할일 목록
- [ ] 중요도별 필터
- [ ] 상태별 필터

### 할일 수정 및 삭제
- [ ] 제목, 내용, 날짜, 중요도, 긴급도, 상태 수정
- [ ] 삭제 (확인 팝업 후 삭제)
- [ ] 클릭 한 번으로 완료 처리
