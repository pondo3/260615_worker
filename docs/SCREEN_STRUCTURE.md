# SCREEN_STRUCTURE.md - 화면 구성 설계서

## 전체 화면 목록

```
로그인 전 접근 가능:
  /login          → 로그인 페이지
  /register       → 회원가입 페이지

로그인 후만 접근 가능:
  /dashboard      → 대시보드
  /tasks          → 할일 전체 목록
  /tasks/today    → 오늘 할일
  /tasks/tomorrow → 내일 할일
  /tasks/overdue  → 지난 미완료 할일
  /schedules      → 스케줄 (이번 주/월별)
  /experiments    → 테스트 기록 목록
  /experiments/[id]         → 테스트 상세
  /experiments/[id]/edit    → 테스트 수정
  /experiments/new          → 새 테스트 등록
  /analysis       → 테스트 결과 분석
  /settings       → 설정
```

---

## 화면 1: 로그인 페이지 (/login)

```
┌─────────────────────────────┐
│         My Work Manager     │
│                             │
│  이메일 [________________]  │
│  비밀번호 [_____________]   │
│                             │
│  [ 로그인 ]                 │
│                             │
│  계정이 없으신가요?          │
│  → 회원가입                 │
└─────────────────────────────┘
```

**포함 요소:**
- 이메일 입력창
- 비밀번호 입력창
- 로그인 버튼
- 회원가입 페이지 링크
- 오류 메시지 표시 영역

## 네비게이션 구조

```
상단 헤더:
  왼쪽: 로고/사이트명
  오른쪽: 사용자 이름 + 로그아웃 버튼

사이드바 또는 상단 메뉴:
  - 대시보드
  - 할일 (드롭다운: 오늘/내일/전체/지난 미완료)
  - 스케줄
  - 테스트 기록
  - 분석
  - 설정
```

## 화면 흐름도

```
[로그인 페이지]
      ↓ 로그인 성공
[대시보드]
   ├── → [오늘 할일]
   │        ├── [할일 추가 모달]
   │        └── [할일 수정 모달]
   ├── → [스케줄]
   │        └── [일정 추가 모달]
   ├── → [테스트 목록]
   │        ├── [새 테스트 등록]
   │        └── [테스트 상세]
   │               └── [테스트 수정]
   ├── → [분석]
   └── → [설정]
```
